# PhyloClimb（仮）ゲーム企画書

## 1. 企画概要

グラップル登攀アクションとローグライク的な進化システムを組み合わせた、高難易度配信向けゲーム。
**「Getting Over It」的な一発逆転の緊張感**と**「進化図鑑を埋める楽しさ」**を融合し、短時間でクリアできる低価格インディーゲームとして、Steam配信者層をメインターゲットに展開する。

**タイトル候補**: PhyloClimb
**ジャンル**: グラップル登攀アクション × ラン制進化（揮発） × 進化図鑑（永続）
**価格**: ¥490
**想定プレイ時間**: 3〜8時間

---

## 2. 背景・目的

### 2.1 背景

- **「Getting Over It」系の高難易度登攀ゲームは配信映えするが、リトライ性が低く飽きやすい**
- **ローグライク要素（育成・コレクション）を組み合わせることで、リプレイ性を向上**
- **低価格・短時間クリアで、配信者が「1回の配信で完結」できる設計を狙う**a

### 2.2 目的

- **商業販売（Steam）での収益化**を目的とした個人開発プロジェクト
- 配信者に取り上げられやすいゲームデザインによる拡散を狙う
- 高難易度×コレクション要素の組み合わせによる、新しいプレイ体験の提供

## 4. 現状の課題と解決策

### 4.1 「Getting Over It」系ゲームの課題

| 課題                           | 影響                           |
| ------------------------------ | ------------------------------ |
| **繰り返しプレイが単調**       | クリア後のモチベーション低下   |
| **理不尽な難易度が敬遠される** | 新規プレイヤーが離脱しやすい   |
| **成長感の欠如**               | 進歩が見えにくく、挫折しやすい |

### 4.2 本企画での解決策

| 課題                           | 解決策                                                                   |
| ------------------------------ | ------------------------------------------------------------------------ |
| **繰り返しプレイが単調**       | **進化図鑑システム**：毎回異なる進化ルートで新しい発見がある             |
| **理不尽な難易度が敬遠される** | **学習可能なギミック設計**：偽フック点、周期風など、知識で攻略できる設計 |
| **成長感の欠如**               | **永続的な図鑑埋め**：ラン制でリセットされても、図鑑は永続的に蓄積       |

---

## 5. 体験の柱（コアゲームデザイン）

### 5.1 4つの核心体験

1. **グラップルで登る気持ちよさ**
   - 射出→ヒット→引き寄せ/振り子動作の流れるような操作感

2. **落下の緊張感**
   - 高さを失う＋**落下距離に比例したダメージ**
   - "もう一回のミスで終わる"状況が配信映えする

3. **進化の選択**
   - ラン中だけ強くなる／死亡でリセット
   - どの進化ルートを選ぶかの戦略性

4. **進化図鑑が埋まる快感**
   - 樹形図で永続コレクション
   - 死んでも知識が積み上がる感覚

## 6. ゲームシステム詳細

### 6.1 クリア構造

```
【初回クリア目標】
└─ 1回の登頂でエンド（到達点が明確で配信向き）

【クリア後のやりこみ要素】
├─ タイムアタック
├─ 縛りプレイ（進化禁止/回復縛り/図鑑埋め縛り等）
└─ 進化図鑑100%コンプリート
```

### 6.2 コアループ（プレイの流れ）

```
1. 登る（グラップル操作）
   ↓
2. 進化素材/進化トリガを発見・収集
   ↓
3. その場で進化分岐（樹形図のどこに伸ばすか選択）
   ↓
4. 罠・ミスで落下
   ├─ 高さを失う
   └─ 落下距離でHPが削れる
   ↓
5. 回復チャンスは稀（完全回復はできない）
   ↓
6. 死亡/リトライで進化（育成結果）は消える
   ただし進化ツリー図鑑は残る（知識が積み上がる）
   ↓
7. 学習したルートと進化で再挑戦 → 登頂成功
```

## 7. システム仕様の骨格

### 7.1 落下ダメージ＆回復

| 要素             | 仕様                                                                                                         |
| ---------------- | ------------------------------------------------------------------------------------------------------------ |
| **HP**           | 最大100                                                                                                      |
| **落下ダメージ** | 距離に応じて増加（小:5-10 / 中:20-35 / 大:50-即死級）                                                        |
| **回復**         | - 1ステージ内に最大2回<br>- 1回につき+25回復<br>- **完全回復はできない**（常にジリ貧の緊張感）               |
| **重要ポイント** | 落下＝「戻される」だけでなく「体力も削れる」ため、<br>**"もう一回のミスで終わる"**状況が生まれ、配信映えする |

### 7.2 進化（育成）—ラン制

- **死亡/リトライで全リセット**
- **進化は樹形図で分岐**（現実の進化っぽい系統）
- **進化の効果は"登攀に直結"するものに限定**（育成が主役になりすぎない）

**進化例（登攀系）**:

- **前肢強化**: グラップル射程 +%、引き寄せ強度 +%
- **尾の安定**: 空中制御 +%、着地ブレ軽減
- **粘着器官**: 壁接触時のスリップ軽減（万能にしない）

### 7.3 図鑑（永続）

- **解放した進化ノードが樹形図に永続記録**
- 図鑑は「埋めるほど嬉しい」だけで、**恒久強化にしない**のが無難
  （難しさの価値が壊れにくい）

## 8. "鬼畜ギミック"の入れ方（理不尽ではなく学習）

| ギミック       | 設計意図                                       |
| -------------- | ---------------------------------------------- |
| **偽フック点** | 掴めるが、一定荷重で外れる（知識ゲー）         |
| **周期風**     | 風周期を覚えると楽になる（実況のネタになる）   |
| **脆い足場**   | 乗ると音がしてから崩れる（気づける余地がある） |

→ **「理不尽」ではなく「知識で攻略できる」設計により、学習の価値を高める**

## 9. MVP（プロトタイプ）—最小の縦切り

### 9.1 最初に作るべき最小要素

```
1. グラップル移動（最重要）
   └─ 射出 → ヒット判定 → ロープ拘束 → 引き寄せ/振り子

2. 落下ダメージ（HP）
   └─ 落下距離計測 → ダメージ → 画面演出

3. 回復ポイント（低頻度）
   └─ 例：1ステージ内に2個だけ

4. 進化ツリー（小さく）
   └─ まずは7ノード程度（2〜3分岐）
   └─ 死亡でリセット

5. 図鑑UI（樹形図表示）
   └─ 解放済みノードを永続表示

6. 短い縦ステージ（30秒〜2分）
   └─ 罠3種だけ（偽フック/風/脆足場）
```

### 9.2 初期の数値設定（仮置き）

| パラメータ       | 数値                              |
| ---------------- | --------------------------------- |
| **HP**           | 100                               |
| **回復**         | 最大2回/ステージ、1回+25          |
| **落下ダメージ** | 小:5-10 / 中:20-35 / 大:50-即死級 |

## 13. リスクと対策

| リスク                              | 影響度 | 対策                                                                      |
| ----------------------------------- | ------ | ------------------------------------------------------------------------- |
| **Phaser3でのグラップル実装が困難** | 高     | 早期にプロトタイプを作成し、実装可能性を検証。困難な場合はUnityに切り替え |
| **高難易度すぎてプレイヤーが離脱**  | 高     | 早期テストプレイで難易度曲線を調整。段階的な学習カーブを設計              |
| **進化システムが複雑すぎる**        | 中     | 初期は7ノード程度の小規模ツリーで検証。複雑化しすぎない設計               |
| **配信者に注目されない**            | 中     | ビジュアル・演出の強化。配信者向けキー配布施策の実施                      |
| **予算オーバー**                    | 低     | 無料素材を優先活用。有料購入は厳選し、10,000円の上限を厳守                |
| **開発期間の長期化**                | 中     | MVPを最優先で完成させ、段階的リリースを検討（Early Access等）             |

## 17. まとめ

本企画は、**「Getting Over It」的な緊張感**と**「ローグライク的なリプレイ性」**を融合した、配信向け高難易度アクションゲームです。

- **低価格（¥490）・短時間（3-8h）** で、配信者が取り上げやすい
- **ラン制進化**により、毎回異なる体験を提供
- **永続的な図鑑システム**で、死んでも進歩がある達成感
- **学習可能な難易度設計**で、理不尽さを排除

個人開発プロジェクトとして、**技術検証を早期に行いながら、MVP→段階的拡充**の開発フローで、リスクを抑えつつ高品質なゲーム体験を実現します。
